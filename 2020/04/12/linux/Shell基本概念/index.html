# shell

Shell 是指「提供使用者使用介面」的軟體，通常指的是命令列介面的解析器 (Command Line Interface, CLI)

電腦是由硬體和軟體所構成的，而主要負責運算的部分是作業系統的核心 - Kernel。當使用者下了命令之後，Kernel 就會接收這個命令並且再交由 CPU 進行處理。那 Kernel 是如何接收這些命令的呢？

可以看到下面這張圖，使用者和 Kernel 會靠著 Shell 作為一個使用者的介面來進行溝通，也就是說使用者下達了命令後，Shell 會將這些命令轉成 Kernel 可以理解的程式碼，再傳送給 Kernel 好讓 Kernel 可以正確地控制硬體工作。

# BASH(Bourne Again SHell)

  • 是一個為 GNU 項目編寫的 Unix shell
  • 是大多數 Linux 系統預設的 shell
  • 是 Mac OS X v10.4 預設的 shell
  • 也被移植到了 Microsoft Windows 上的 Cygwin 中，以實現 windows 的
  • POSIX 虛擬介面

BASH 是 Linux 預設的 Shell，雖然 Shell 有很多種，但是 BASH 會被作為 Linux 預設的 Shell 是因為他的功能非常強大，下面列出幾個主要的優點 :

  1. 命令紀錄 (history) 相信只要有碰過 Linux 或是 Windows 的 cmd 都會知道，只要命令輸入並執行過，下次只要按 向下 和 向上 鍵就可以找回輸入過的指令。這些輸入過的指令會先被記錄在 cache 中，當 Shell 被關閉時就會被寫入到 ~/.bash_history 中，不過這個檔案只會保持一定數量的命令紀錄。

  2. 命令補全 當輸入命令時，可以只輸入指令或是檔案的前幾個字，接著按下 tab 鍵就可以將指令或是檔案補齊。

  3. 命令別名 有些常用的命令可能比較長，這時候就可以使用 alias 指令來設定，將命令設成一個別名，之後只要輸入別名就可以執行了。

下面我們來實際操作一下，將 ls -a 這個指令設成 la ，接著只要輸入 la 就可以達成 ls -a 的結果。此外，如果要知道有哪些別名的命令可以直接輸入 alias 來查看。


4. 程式化腳本 (Shell Script) 可以將多個指令寫在一個檔案中，達成連續下指令的動作。這可以大大簡化輸入常用指令的時間，有點類似於程式語言的作法。

  1. 萬用字元 (Wildcard) 可以透過萬用字元 * 結合指令來進行操作。

# 查詢指令說明文件

想要查詢指令的說明文件可以使用 help 或 man 指令。但是在介紹這些指令前，要先知道指令有分為 內部指令 (builtin shell command) 和 外部指令 。內部命令是指本身已經內建於 Shell 的指令，而外部指令是額外安裝的應用，通常會放在 /bin 、 /usr/bin 、 /sbin 、 /usr/sbin 等等。

  • OR || 的用法與 && 相反，如果前一個指令執行失敗就會執行第二個，反之如果第一個執行成功就不會執行的二個。

# 指令換行

有時候指令會很長，雖然很長並不會有問題，但是對自己來說卻很不好讀。這時候就可以在要換行的時候輸入 \ + Enter 來跳離 Enter 鍵，此時游標就會跳到下一行就可以繼續輸入了。

# Shell 命令語法 :

# 命令三元素 :

  • Command : 命令
  • Option : 選項
  • Argument : 額外參數

# 選項 or 參數格式 :

  • [] : 選擇性欄位 (可有可無)
  • ... : 多個同性質的選項 or 參數
  • {a|b|c} : a,b,c : 三選一
  • <filename> : 變數
  • -a,--all : 長參數與短參數 (當參數超過 2 字元以上會定義為長參數)(長參數後面若需要加值必須加上 =)

# --help : 查詢指令用法

# man : 使用者手冊

# pinfo : 與 man 相似

# clear : 清除终端屏幕 (快捷鍵 : Ctrl+L)

# passwd : 變更密碼

# whoai : 確認使用者

# date : 現在的時間 (參數很重要，例如 : date +% R)

# file : 顯示文件類型

# cat : 串接或輸出檔案內容

# head : 可以查看文件開頭內容，常用的參數為 - n 用於自訂顯示行數，默認為 10

# tail : 可以查看文件最後內容，常用的參數為 - n 用於自訂顯示行數，默認為 10

# wc : 計算指定檔案的行數、字數，以及位元組數

# 快速完成 :

# pas [tab][tab] : 找到開頭為 pas 的所有指令

# pass [tab] : 直接生成開頭為 pass 的指令

# history : 出現所有曾經打過的所有指令 (有編碼)

# !number : 為使用 history 編號下的指令

# !pass : 使用 history 曾經使用過的指令 (可以不用全部打完)
