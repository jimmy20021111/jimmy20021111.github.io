# 系統服務與查看

傳統上的 Linux 都是靠 System V 的 init 來啟動各項系統服務，而後來新的 Systemd 出現之後，許多主流的 Linux 發行版都改用 Systemd 來管理系統服務，新的 Systemd 跟傳統 System V 的 init 相比，開機速度更快、效能更好、具有相依性檢查功能。

# systemd 基本服務管理

在 Systemd 中每一個系統服務就稱為一個服務單位（ unit ），而服務單位又可以區分為 service 、 socket 、 target 、 path 、 snapshot 、 timer 等多種不同的類型（ type ），我們可以從設定檔的附檔名來判斷該服務單位所屬的類型，最常見的就是以 .service 結尾的系統服務，大部分的伺服器都是屬於這種。

如果要管理 Systemd 中的各種服務，可以使用 systemctl 這個指令，配合各種操作指令來進行各種操作。

# ststemctl 指令


option :

  • -a 顯示所有單位
  • -f 覆蓋任何衝突的符號鏈接
  • -H 設定要連接的主機名
  • -M 設定要連接的容器名
  • -n 設定要顯示的日誌行數
  • -o 設定要顯示的日誌格式
  • -q 靜默執行模式
  • -r 顯示本地容器的單位
  • -s 設定要發送的進程訊號
  • -t 設定單元類型
  • --help 顯示幫助訊息
  • --version 顯示版本訊息

command :

  • start 啟動服務
  • stop 停止服務
  • restart 重啟服務
  • enable 設定服務開機自啟
  • disable 取消服務開機自啟
  • status 查看服務狀態
  • list 顯示所有已啟動服務

# 列出 systemd 所有服務

若想要列出所有 已啟動 的服務，可以使用 list-units 操作指令：

  • UNIT : Systemd 服務單位（ unit ）名稱。
  • LOAD : 該服務單位設定檔是否有被 Systemd 載入至記憶體中。
  • ACTIVE : 是否已經正常啟動。
  • SUB : 更詳細的狀態說明，值會因為不同服務有所不同。
  • DESCRIPTION : 關於此服務的簡單說明。

事實上 systemctl 的預設動作就是列出所有已啟動的服務，所以也可以省略 list-units

# 確認服務執行的狀態

  • 執行中的服務 -> active
  • 開機程序中執行的服務 -> enable

# 查看單一服務狀態

  • systemctl is-active name.type
  • systemctl is-enabled name.type
  • systemctl is-failed name.type

# 系統服務控指

# runtime (控制正在記憶體中的狀態)

  • start
  • stop
  • reload
  • restart

# startup (重新啟動會執行的服務)

  • enable
  • disable
