# 基本指令用法

# ps

用來顯示目前進程的狀態，類似 windows 的工作管理員。


option :

  • -A 列出所有的進程
  • -w 顯示加寬可以顯示較多的資訊
  • -au (u) 顯示較祥细的資訊
  • -aux 顯示所有包含其他使用者的進程
  • -e 顯示所有進程，環境變量
  • -f 全格式
  • -h 不顯示標題
  • -l 長格式
  • -w 寬輸出
    範例
  • USER: 使用者登入名稱
  • PID：處理程序 ID
  • % CPU：佔用的 CPU 使用率
  • % MEM：佔用的記憶體使用率
  • VSZ：佔用的虛擬記憶體大小
  • RSS：佔用的記憶體大小
  • TTY: 處理程序的控制工作站
  • STAT：行程的狀態：
    • D: 無法中斷的休眠狀態（通常是 IO 的進程）
    • R: 正在執行中
    • S: 靜止狀態
    • T: 暫停執行
    • Z：不存在但暫時無法消除
    • W：沒有足夠的記憶體分頁可分配
    • <: 行程的優先順序
    • N：行程的低優先順序
    • L：有記憶體分頁分配並鎖在記憶體內（即時系統或鍵盤 AI/O）
    • X : 掛起的行程
    • s : 進程的領導者（在它之下有子進程）
    • + : 位於後台的進程組
  • START：行程開始時間
  • TIME: 處理程序的執行時期總計
  • COMMAND: 所執行的指令

# jobs

查看目前終端機有多少在背景運行的進程。

  • jobs 指令執行的結果， ＋ 表示是一個目前的作業， - 減號表示是一個目前作業之後的一個作業。

  • jobs -l 選項可顯示所有任務的進程編號 pid

  • jobs 的狀態可以是 running ， stopped ， terminated 。但如果任務進程被終止了（ kill ），當前的終端環境中也就刪除了任務的進程標識；也就是說 jobs 命令顯示的是當前 shell 環境中後台正在運行或者被掛起的任務進程信息

範例

# &

用在一個指令的最後，可以把這個指令轉換成後台運行的任務進程。

範例

# fg(foregroud)

將後台任務進程調至前台繼續執行，如果後台中有多個任務進程，可以用 fg %num 將選取的任務進程調至前台，默認為 + 的工作為主。

範例

# bg(backgroud)

將掛起的任務進程重新啟動運行，將 Stopped 變為 Running ，如果有多個暫停的任務進程，可以用 bg %num 將選取的任務進程啟動運行，默認為 + 的工作為主。

範例


%num 是透過 jobs 指令查到的後台正在執行的任務的序號（不是 pid ）

# 進程的掛起 (Killed)

# 後台的掛起

  • 在 solaris 中透過 stop 指令執行，透過 jobs 指令查看任務號碼（假設為 num），然後執行： stop %num
  • 在 redhat 中，不存在 stop 命令，可透過執行命令 kill -stop PID ，將進程掛起

# 前台的掛起

ctrl + z ：可以將一個正在前台執行的任務放到背景運行，並且掛起

# 掛起進程重新運行

  • 通過 bg %num 即可將掛起的任務進程的狀態由 stopped 改為 running ，仍在後台運行
  • 通過 fg %num 即可將掛起的任務進程轉為前台執行

# 進程的終止

# 後台程序的終止

  • 方法一： 透過 jobs 指令查看任務號（假設為 num ），然後執行： kill %num
  • 方法二： 透過 ps 指令查看任務的進程號（PID，假設為 pid），然後執行： kill pid

# 前台程序的終止

執行 ctrl+c 即可終止前台執行任務行程

# killall

直接使用程式的名稱來指定要中止的程式，這樣只要知道程式名稱即可直接使用，將符合名稱的所有程序終止掉。

killall 範例

# pkill

用於殺死一個進程，與 kill 不同的是它會殺死指定名字的所有進程，類似於 killall 命令。

pkill 範例

# pgrep

可以搭配 pgrep 這個指令去查詢名稱的 PID

  • -d : 定義多個進程之間的分隔符號，如果不定義則使用換行符號。
  • -P : 根據父行程 PID，找出所有字進程的 pid
  • -n : 表示如果程式有多個進程正在運行，則僅查找最新的，即最後啟動的。
  • -o : 表示如果該程式有多個進程正在運行，則只查找最老的，即最先啟動的（多個進程時即父進程 PID）。
  • -G : 其後跟著一組 group id，該指令在搜尋時，只考慮 group 清單中的進程。
  • -u : 其後跟著一組有效使用者 ID (effetive user id)，在搜尋時，只考慮該 effective user 清單中的進程。
  • -U : 其後跟著一組實際使用者 ID (real user id)，在搜尋時，只考慮該 real user 清單中的流程。
  • -x : 表示進程的名字必須完全匹配，以上的選項均可以部分匹配。
  • -l : 將不僅列印 pid, 也列印進程名。
  • -f : 一般與 - l 合用，將列印進程的參數。

# 程序的監控

系統 CPU 資源


** 系統記憶體資源

# uptime

顯示系統總共運行了多長時間和系統的平均負載分別為 1,5,15 分鐘。

# w

執行指令可得知目前登入系統的使用者有哪些人，以及他們正在執行的程式。

  • -f 開啟或關閉顯示使用者從何處登入系統。
  • -h 不顯示各欄位的標題資訊列。
  • -l 使用詳細格式列表，此為預設值。
  • -s 使用簡潔格式列表，不顯示使用者登入時間，終止機階段作業和程式所消耗的 CPU 時間。
  • -u 忽略執行程式的名稱，以及程式消耗 CPU 時間的資訊。
  • -V 顯示版本資訊。

範例

# top

觀察程式運行所花的記憶體、CPU 還有各種其他資訊。就像 Windows 系統中的工作管理員，可以監看目前所有程式的執行狀況。
