# 日誌檔的作用

日誌文件用於記錄 linux 系統的各種運行資訊的文件，相當於 linux 主機的日記，不同的日誌文件記載了不同類型的信息，如 Linux 內核訊息、用戶登入事件、程式錯誤等。. 日誌檔案對於診斷和解決問題很有幫助，因為 linux 運行的程式通常把系統的訊息和錯誤寫入對應的日誌文件，這樣系統可以有據可查，此外，當主機遭受攻擊時，日誌文件還可以幫助尋找攻擊者留下的痕跡。

# 日誌檔的分類

  1. 核心及系統日誌：這種日誌資料由系統服務 rsyslog 管理，根據其主設定檔 /etc/rsyslog.conf 中的設定決定將核心訊息及各種系統程式訊息記錄到什麼位置。系統中大部分的程式會把自己的日誌檔案交由 rsyslog 管理，因而這些應用程式使用的日誌記錄格式都很相似。

  2. 使用者日誌：用於記錄 Linux 系統使用者登入及登出系統的相關資訊，包括 使用者名稱 、 登入的終端 、 登入時間 、 來源主機 、 正在使用的進程操作 等。
  3. 程序舊志：有些應用程式會選擇獨立管理一份日誌檔案，而不是交給 rsyslog 服務管理，用於記錄本程式運行過程中的各種事件資訊。於這些程式只負責管理自己的日誌檔案，因此不同程式所使用的日誌記錄格式也會有較大的差異。

# 日誌檔案的位置

Linux 系統本身和大部分伺服器程式的記錄檔預設放在 /var/log/ 下。部分程式共用一個日誌檔案，一部分程式使用單一日誌檔案。而有些大型伺服器程式日誌由於日誌檔案不只一個，所以會在 /var/log/ 目錄中建立相應的子目錄來存放日誌檔案，這樣既保證了日誌檔案目錄的結構清晰，又可以快速定位日誌檔案。有一部分日誌檔案只有 root 使用者才有權限讀取，這保證了相關日誌資訊的安全性。

# 常見的日誌檔案及查看方式

  • /var/log/boot.log : 記錄系統啟動時的日誌訊息，如果系統啟動之後有什麼異常可以查看該文件信息。
  • /var/log/message : 核心訊息及各種應用程式的公共日誌資訊，包括啟動、I/O 錯誤、 網路錯誤。
  • /var/log/cron : 系統的排程，週期性計劃任務產生的時間訊息。
  • /var/log/dmesg : 引導過程中的各種時間資訊。
  • /var/log/ maillog : 進入或發出系統的電子郵件活動。
  • /var/log/lastlog : 每個使用者最近的登入事件。
  • /var/log/secure : 使用者認證相關的安全事件訊息。
  • /var/log/wtmp : 每個使用者登入登出及系統啟動和停機事件。
  • /var/log/btmp : 失敗的、錯誤的登入嘗試及驗證事件。
  • /var/log/secure : 一般用來記錄安全相關的訊息，紀錄最多的是哪些使用者登入伺服器的相關日誌，如果檔案很大，表示有人在破解你的 root 密碼。

# 查詢指令

  • who : 報告目前登入系統的每個使用者的訊息，包括使用者名，終端，登入日期，遠端主機。
  • w : 顯示目前作業系統的每個使用者及其遠端所執行的進程訊息。
  • last : 用於查詢成功登入系統的使用者記錄，最近的登入情況在最前面。透過 last 指令可以及時掌握 Linux 主機的登入情況，若發現未經授權的使用者登陸過，則表示目前主機可能已被入侵。
    • -a : 把從何處登入系統的主機名稱或 IP 位址，顯示在最後一行
    • -d : 將 IP 位址轉換成主機名稱
    • -f : 指定記錄檔
    • -R : 不顯示登入系統的主機名稱或 IP 位址
    • -x : 顯示系統關閉，重新開機，以及執行等級的改變等
    • -n : n 代表數字，表示最近 n 次登入的記錄
  • lastlog : 用於顯示系統中所有使用者最近一次登入訊息。
  • lastb : 用於顯示使用者錯誤的登入清單，此指令可以發現系統的登入異常。如登入的使用者名稱錯誤，密碼不正確等情況都會記錄在案，登入失敗的情況屬於安全事件，因此消失可能有人在嘗試破解密碼。除了使用 lastb 指令以外，可以直接從安全性日誌檔案 /var/log/secure 中取得相關資訊。

option :

  • -T , --tcp : 使用流連接 (TCP)
  • -d , --udp : 使用數據報 (UDP)

  • -f , --file <file> : 紀錄特定的文件
  • -h , --help : 顯示幫助文本並退出

  • -P , --port <port> : 使用指定的 UDP 端口。默認的端口號是 514
  • -p , --priority <prio> : 指定輸入消息的優先級，優先級可以是數字或者指定為 "facility.level" 的格式。比如：" -p local3.info "local3 这个设备的消息级别为 info。默认级别是"user.notice"
  • -s , --stderr : 輸出標準錯誤到系统日誌。
  • -t , --tag <tag> : 指定標記紀錄
  • -u , --socket <socket> : 寫入指定的 socket，而不是到内置系统日誌例程。
  • -V , --version : 輸出版本信息並退出

# 步驟範例

# System journal

日誌本身是 systemd 管理的系統服務，完整名稱為 systemd-journald.service 。它會根據從核心、使用者程序、標準輸入和系統服務錯誤收到的記錄資訊，維護結構化的索引記錄，藉以收集和儲存記錄資料。 systemd-journald 服務預設處於開啟狀態。

# 將日誌設為永久

依預設，日誌在 /run/log/journal/ 中儲存記錄資料。由於 /run/ 目錄具有易失本性，因此，在重新開機時會遺失記錄資料。若要永久儲存記錄資料， /var/log/journal/ 目錄必須存在且具有正確的擁有權和許可權，如此， systemd-journald 服務便可在其中儲存其資料。 systemd 將為您建立該目錄，如果您執行以下操作，它將會切換到永久記錄︰
